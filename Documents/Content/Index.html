📋 System Overview

The VBA-Web platform is a comprehensive manufacturing execution system built with Excel VBA. It handles production operations across four specialized applications with real-time SAP ERP integration, offline-first design, and complete audit trail capabilities.

✨ Key Capabilities

  • Online/Offline operation with eventual consistency
  • Real-time SAP GUI Scripting integration
  • 7 SAP transactions automated (COR6, COR3, MB52, MIGO, etc.)
  • Local spreadsheet + network backup architecture
  • FIFO batch sequencing for reliable processing
  • Quality control & non-conformity tracking
  • Real-time inventory synchronization
  • Comprehensive audit trail with operator tracking
  • Centralized error handling & recovery
  • Multi-furnace parallel entry (up to 4 channels)

🔒 Security & Compliance

Security Features

  • Authentication: Windows AD integration (inherits file share permissions)
  • User Tracking: Automatic capture of operator via Environ$("UserName")
  • Audit Trail: Complete timestamp + operator logging for every transaction
  • Data Integrity: Append-only after SAP posting (no deletions)
  • Network Security: File share backups with access controls
  • SAP Integration: Inherits user's SAP authorization level

Data Protection Strategy

  • At Rest: Relies on file system permissions (no encryption)
  • In Transit: SAP GUI Scripting over secure session
  • Backup: Network share with automatic redundancy
  • Recovery: Offline queuing ensures no data loss

Error Handling & Recovery

  • Centralized StdErrorHandler for all errors
  • Offline mode fallback when SAP unavailable
  • FIFO queue ensures ordered batch processing
  • Automatic retry logic for transient failures
  • User notifications with actionable error messages

📊 Data Models

Universal Database Schema

All applications follow a consistent data model:

Sheet1: Transaction Log

  • Column A: Identifier (Furnace/Timestamp)
  • Column B: Timestamp (auto-NOW())
  • Column C: Pallet/Container ID
  • Column D: SAP Process Order
  • Column E: Item Number
  • Column F: Operator ID
  • Column G+: App-specific fields (batches, measurements, etc.)

Sheet2: Configuration

  • Online/Offline mode flag
  • Last used form values (furnace, item, operator)
  • Cached material specifications
  • Equipment assignments & furnace data
  • Dimension ranges (mins/maxes)

Optional: SandTrans Sheet

  • Material number, batch, quantity
  • Source location (0015/0100/0400)
  • Destination furnace
  • Status flag (Pending/Completed)

🔄 Key Workflows

Standard Transaction Flow (All Apps)

  1. User Opens Application: Load config from Sheet2
  2. Determine Mode: Check online/offline status
  3. Display Form: Show appropriate UserForm with defaults
  4. User Enters Data: Input all required fields
  5. Validation: checkInputs() validates all entries
  6. Save Locally: Append row to Sheet1
  7. Create Backup: Copy to network share
  8. If Online: Execute SAP transactions (COR6, MIGO, etc.)
  9. If Offline: Queue for batch processing
  10. Completion: Display success/error, prompt to continue

Offline-to-Online Sync

  1. User works in offline mode (form displays yellow background)
  2. Transactions saved to local sheet + SandTrans queue
  3. When SAP available, supervisor initiates batch processing
  4. FIFO queue processed sequentially in order
  5. Each transaction: COR6 post → MIGO → HU creation (if applicable)
  6. All transactions synced to remote backup
  7. Queue marked complete
