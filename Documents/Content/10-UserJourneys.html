🔥 Bake Application Journeys

Journey 1: Serialized Bake Load

  1. Operator opens BakeApplication.xlsm
  2. Select furnace and check online/offline status
  3. Enter pallet card number (triggers batch history lookup)
  4. Enter batch numbers (1-15) with duplicate detection
  5. Enter bake hours total
  6. Select hamper and strapping item
  7. Optional: Add return hamper & waffle boards
  8. Click Submit
  9. System validates all inputs
  10. Save to local sheet
  11. Create network backup
  12. If online: Post COR6 to SAP & create HU
  13. Display completion message

Journey 2: Sand Transfer

  1. Click "Sand Transfer" button
  2. Enter sand material item number
  3. Enter sand batch number
  4. If online: Query MB52 for stock availability
  5. System displays locations (0015, 0100, 0400)
  6. Select source location
  7. Enter transfer quantity (kg)
  8. Confirm transfer details
  9. Execute MIGO movement (if online) or queue (if offline)

📦 HV Pack Application Journey

Complete Packing Workflow

  1. Operator opens HVPack.xlsm
  2. Enter finished item number
  3. System auto-looks up description (if online)
  4. FOR EACH BATCH (1-8):
    • Enter batch number
    • System detects duplicates
    • Enter measurements (length, bow)
    • Click "Complete Tube"
    • Save single tube record
    • If online: Post individual COR6
  5. After last tube: Display hamper configuration options
  6. Select hamper from dropdown
  7. Enter quantity packed
  8. System calculates strapping qty (24× hamper_qty)
  9. Optional: Add return hamper & qty
  10. Optional: Add waffle boards
  11. Click "Submit Hamper"
  12. Validation of all hamper fields
  13. Build material application array
  14. Save to local sheet
  15. Create network backup
  16. If online: COR6 + MIGO + HUMO + Label print
  17. Display completion
  18. Clear form for next pallet

🎣 Lamp Rod Draw Journey

Production Draw & Sand Management

  1. Operator opens LampRodDraw.xlsm
  2. Select furnace (FR11-14)
  3. Enter process order number
  4. System queries COR3 for item details (if online)
  5. Display item description & specifications
  6. Enter production data:
    • Pallet card
    • Technician ID
    • Quantity produced
    • Labor hours
  7. Optional: Record scrap/non-conformity (0-4 entries)
  8. For each scrap entry: Select reason code
  9. Click Submit
  10. Validate all inputs
  11. Save locally
  12. Create backup
  13. If online: Post COR6 confirmation
  14. Update equipment hours
  15. If sand needed: Execute sand transfer
  16. Display completion

📐 Serialized Material Journey

Precision Tube Entry Workflow

  1. Operator opens SerializedMaterial.xlsm
  2. Select furnace channel (Green/Red/Blue/Yellow page)
  3. Enter pallet card number
  4. Enter process order
  5. System auto-looks up item description & specs
  6. MEASUREMENT ENTRY:
    • Enter OD1 measurement → Auto-validate range
    • Enter OD2, OD3, OD4 (same validation)
    • Enter WALL1-WALL8 measurements
    • System auto-formats to 2 decimals
    • Color feedback: Green (valid) / Red (out of range)
  7. Enter bow/curvature value (decimal only)
  8. Enter exact length
  9. QUALITY CHECK:
    • Toggle non-conformity flag if defective
    • If defective: Select from 13 defect codes
    • If certain codes: Measurements auto-hidden
    • Select material routing (Webb/Scrap-D/Plate/Trash)
  10. Click Submit Tube
  11. Validation of all fields
  12. Generate tube serial number
  13. Save to local sheet
  14. Create network backup
  15. If online:
    • Post COR6 production confirmation
    • Apply dimensions to SAP
    • Display tube serial in alert
    • Update backup
  16. Ready for next tube

⚡ Offline-to-Online Recovery

Batch Processing After SAP Outage

  1. During SAP outage: Operator works in offline mode
  2. Transactions saved to local sheet + SandTrans queue
  3. Form background shows yellow (offline indicator)
  4. When SAP available: Supervisor initiates batch processing
  5. System processes FIFO queue sequentially
  6. For each queued transaction:
    • Retrieve transaction from queue
    • Execute COR6 posting
    • Apply material movements (MIGO)
    • Create HU if applicable
    • Mark queue item completed
  7. All transactions synced to remote backup
  8. Queue cleared
  9. Operations resume normal mode
